## TEMPLATES FOR QUICK REPLIES

### If someone asks "How is this different from DSPy?"
```
Good question. DSPy optimizes prompts through automated prompt search/compilation.
GSEP takes a different approach — biological evolution with safety constraints.

Key differences:
1. GSEP wraps your existing agent (not a new framework)
2. Continuous evolution (not one-time optimization)
3. Safety-first: immutable DNA, content firewall, immune system
4. Per-user adaptation (epigenomes)

They solve different problems. DSPy = initial prompt optimization.
GSEP = ongoing prompt evolution + safety.
```

### If someone asks "Does this actually work in production?"
```
Honest answer: we have proof-of-value benchmarks showing +16% quality
improvement, and the architecture is production-ready (1600+ tests,
canary deployments, auto-rollback).

But we're early — launched a week ago. We need real-world production
usage to validate at scale. That's why we're MIT-licensed and looking
for early adopters to stress-test it.

The proof-of-value runner is built into the SDK so you can measure
improvement on YOUR specific use case.
```

### If someone says "This is over-engineered"
```
Fair criticism. The full system is comprehensive, but you don't need
all of it. GSEP has 4 configuration presets:

- minimal: just basic evolution (4 flags)
- standard: production-ready (13 flags)
- conscious: full intelligence (20 flags)
- full: everything (21 flags)

Most people should start with `minimal` and add features as needed.
The complexity is opt-in.
```

### If someone says "I don't trust self-modifying prompts"
```
I 100% agree that safety is the critical question.

That's why the first thing I built was the constraints, not the evolution:

1. Layer 0 (identity/ethics) is SHA-256 locked — CANNOT mutate
2. Every mutation passes 4 validation gates before deployment
3. Canary deployments test on 10% traffic first
4. Auto-rollback if fitness drops
5. Content firewall scans all input (57 patterns)
6. Immune system checks all output (6 deterministic checks)

The system is designed so the WORST case is "nothing changes"
(mutation rejected), never "something bad happens."
```
